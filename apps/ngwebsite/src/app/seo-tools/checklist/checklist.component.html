Hints, tools and a list of TODO for your web application. Be sure to complete almost all steps

Domain

  • Redirect your users to your domain with http://www. instead of http://mydomain.com
  • Use https if possible
  • Check if your domain is on a blacklist

Assets Optimization

  • Minify HTML output: I don't understand why many websites don't do this but this is a very useful and essential feature to implement.
  • Minify CSS: compress \ minify CSS files. Compress all CSS files in only one file.
  • Minify Javascript files. Include all files and create only one file.
  • Validate HTML: valid HTML helps accessibility.
  • Validate CSS if possible
  • Delete unused CSS parts, classes or files
  • Optimize images: use optimizers for your operative system, use svg or base64_encode for images.
  • Set a resposive size for your images

Improving HTML

  • Use a responsive layout. A website must be viewed correctly with multiple devices.
  • Use microdata to help search engines to find your pages
  • Use AMP HTML to enhance speed and performaces
  • Use structured data and the Structured Data Testing Tool
  • Turn your website in a Progressive Web Application
  • Be sure to have a sitemap for the users
  • Write or generate a privacy policy page. It will be useful for your facebook app, footer and other stuff.
  • Use the alt attribute for every image
  • Add accesskeys for the main sections links
  • Add the title attribute for every link
  • Use semantic HTML
  • Use a correct doctype
  • Backlinks check

Meta tags

  • Title tags with no stop words and a medium legnth (maxi 160\170 characters)
  • Write description tags (from 250 \ max 300 characters).
  • Do not use noindex attribute
  • You can create a favicon
  • Use the canonical link markup
  • Avoid using Black Hat SEO techniques or your domain will be blacklisted!
  • Use If Modified since

Social Networks

  • Create a Facebook app and associate a page for your website
  • Use Open Graph meta tags
  • Use Twitter meta tags

Server side (Backend)

  • Use browser caching
  • Use Nginx instead of Apache if possible
  • Enable URL rewrite
  • Use SEO friendly URLs: not too long or short.
  • Enable Gzip compression if you can and if you have this feature in your hosting plan.
  • Avoid "false 404 errors": a broken link must return 404 status but if your website returns 200 is even worst because the search engine spider and even the users can be confused
  • Generate and update your XML sitemap. Include the sitemap in your robots.txt file and be sure that the link is correct. If you have a blog, you can generate a Google News Sitemap
  • If you are using Wordpress, you can use SEO plugins like Yoast or Jetpack. But remember you need a plugin for every feature so Wordpress will be definetely heavy!
  • Use ETag header
